# خطة التنفيذ الكامل لتشغيل Rose School 2026 على ويندوز

## ملاحظة مهمة
هذه الخطة مبنية على فحص static للكود الحالي، وليست نتيجة تشغيل فعلي داخل Arena لأن Flutter غير متاح هنا. لذلك أي شيء خاص بـ Windows runtime يحتاج اختبار محلي على جهازك بعد كل مرحلة.

---

## وضع المشروع الحالي

### الجاهز أو شبه الجاهز
- الهيكل العام والواجهة الأساسية.
- تسجيل الدخول والصلاحيات الأساسية.
- SQLite كطبقة حفظ محلية.
- التخزين المحلي للصور وQR والمرفقات.
- بطاقة الطالب وتصدير PDF/PNG.
- لوحة الامتحانات والجلاء المدرسي.
- المحاسبة الأساسية مع الأقساط والتبرعات والمساعدات.
- HTML Preview متوافق مع جزء كبير من المنطق.

### غير المكتمل بالكامل وظيفيًا
1. **التقارير الحقيقية**
   - أزرار Excel / CSV / PDF ما زال جزء منها Demo.
2. **النسخ الاحتياطي والاستعادة**
   - الاستعادة ما زالت Demo.
   - لا يوجد Backup Package منظم كامل فعليًا داخل التطبيق.
3. **مسارات التخزين على ويندوز**
   - التخزين الحالي يحتاج توحيدًا في مجلد واضح للمستخدم.
4. **مركز البيانات المحلي**
   - الصفحة الحالية ليست مركزًا كاملاً لإدارة البيانات.
5. **العمليات الإدارية الحساسة**
   - لا يوجد guard بكلمة سر المدير قبل backup / restore / export.
6. **التصدير الخارجي**
   - لا يوجد Excel حقيقي حتى الآن.
   - لا يوجد JSON Snapshot رسمي داخل التطبيق.
7. **اختبارات Windows workflow**
   - لا توجد دورة اختبار محلية موثقة لكل الأبواب بعد البناء.

## الهدف التنفيذي
تحويل المشروع من واجهات قوية + وظائف كبيرة جزئيًا إلى تطبيق ويندوز كامل الاستخدام على جهاز واحد بحيث:
- يحفظ البيانات محليًا بشكل ثابت وواضح.
- يستعيدها ويؤرشفها بشكل آمن.
- يصدر تقارير حقيقية.
- يعمل يوميًا على ويندوز دون الاعتماد على HTML preview.

## المراحل المقترحة

## المرحلة A — توحيد مسارات التخزين المحلي
### الهدف
توحيد الحفظ داخل مجلد واضح مثل:
- `Documents/Rose School 2026/`

### التنفيذ
- إنشاء `lib/services/app_storage_paths_service.dart`
- نقل قاعدة البيانات إلى:
  - `Documents/Rose School 2026/data/rose_school_2026.db`
- نقل الملفات إلى:
  - `Documents/Rose School 2026/files/...`
- إضافة migration آمن من المسارات القديمة.

### النتيجة
- كل البيانات تصبح في مكان واضح للمستخدم.
- أسهل في النسخ والنسخ الاحتياطي.

---

## المرحلة B — مركز البيانات المحلي
### الهدف
تحويل صفحة `النسخ الاحتياطي والاستعادة` إلى **مركز بيانات محلي** حقيقي.

### الأزرار المطلوبة
- إنشاء نسخة احتياطية الآن
- استعادة نسخة احتياطية
- تصدير JSON
- تصدير Excel
- معاينة النسخة
- فتح مجلد البيانات
- فتح مجلد النسخ الاحتياطية
- تحديث الحالة

### المعلومات الظاهرة
- مسار قاعدة البيانات
- مسار الملفات
- مسار النسخ الاحتياطية
- حجم قاعدة البيانات
- عدد النسخ
- آخر نسخة احتياطية
- حالة التخزين

---

## المرحلة C — النسخ الاحتياطي والاستعادة الحقيقيان
### الهدف
بناء Backup Package كامل.

### صيغة النسخة
- `ROSE_BACKUP_YYYY_MM_DD_HHMMSS.zip`

### المحتويات
- database copy
- snapshot JSON
- files/
- manifest.json
- preview html

### الاستعادة
- Replace Restore في المرحلة الأولى
- فحص الملف قبل الاستعادة
- تأكيد من المستخدم

---

## المرحلة D — التصدير الحقيقي
### الهدف
تحويل أزرار التقارير من Demo إلى وظائف فعلية.

### التصديرات المطلوبة
1. `students_export.xlsx`
2. `accounting_export.xlsx`
3. `exam_results_export.xlsx`
4. `data_snapshot.json`
5. `backup_preview.html`

### ملاحظة
- Excel للتقارير فقط.
- JSON للنسخ الاحتياطي والمنطق.
- SQLite يبقى المصدر الأساسي للبيانات.

---

## المرحلة E — حماية المدير
### الهدف
منع الوصول للعمليات الحساسة بدون إعادة التحقق.

### العمليات التي تتطلب كلمة سر المدير
- إنشاء نسخة احتياطية
- استعادة نسخة
- تصدير JSON الكامل
- تصدير Excel الكامل
- فتح مركز البيانات المحلي
- حذف نسخة احتياطية

### التنفيذ
- `lib/dialogs/admin_password_dialog.dart`
- guard helper على العمليات الحساسة.

---

## المرحلة F — اختبار منطق الأبواب على ويندوز
### الهدف
التأكد أن التطبيق لا يكتفي بالبناء بل يعمل فعليًا حسب منطق النظام.

### مصفوفة الاختبار
#### أمانة السر
- إضافة طالب
- تعديل طالب
- حذف طالب
- رفع صورة
- توليد QR
- رفع مرفق
- حذف مرفق

#### الامتحانات
- فتح مادة
- حفظ درجات فصل أول
- حفظ درجات فصل ثانٍ
- تفعيل التدقيق اليدوي
- التدقيق الجماعي
- فلتر المواد غير المدققة
- معاينة الجلاء
- طباعة الجلاء
- طباعة جماعية

#### المحاسبة
- إضافة قسط متعدد القوالب
- إضافة تبرع مادي/عيني
- إضافة مساعدة مادية/عينية
- فرز حسب الطالب/الشعبة

#### الإدارة
- إضافة مستخدم
- تعديل مستخدم
- حذف مستخدم
- حفظ هوية المدرسة
- حفظ أسماء التوقيع

#### مركز البيانات المحلي
- إنشاء Backup
- استعادة Backup
- تصدير JSON
- تصدير Excel
- معاينة Preview

## القرار التنفيذي الحالي

### المرحلة A — توحيد مسارات التخزين المحلي
لأنها الأساس لكل ما بعدها على ويندوز.

بعدها مباشرة:
### المرحلة B + C
