### المرحلة 1 — توحيد مسارات التخزين المحلي (A)

**الملف:** `lib/services/app_storage_paths_service.dart`

**الهدف:** كل التخزين يكون تحت مسار واحد قابل للوصول من أي مكان في التطبيق.

**التنفيذ:**
```
class AppStoragePathsService {
  Future<Directory> get root async         → Documents/Rose_School_edu/
  Future<Directory> get dataDir async      → .../data/
  Future<Directory> get filesDir async     → .../files/
  Future<Directory> get backupsDir async   → .../backups/
  Future<Directory> get reportsDir async   → .../reports/
  Future<Directory> get configDir async    → .../config/
  Future<String> get databasePath async    → .../data/rose_school_2026.db
  Future<Directory> studentFilesDir(id)    → .../files/students/{id}/
  Future<Directory> employeeFilesDir(id)   → .../files/employees/{id}/
}
```

**يتم تعديل:**
- `school_database_service.dart` ← يستخدم `appStoragePaths.databasePath`
- `local_student_file_service.dart` ← يستخدم `appStoragePaths.filesDir`

### المرحلة 3 — الموظفون (ذاتيات الموظفين) (B)

**الملفات:**
- `lib/pages/employees_page.dart`
- `lib/pages/employee_finance_review_page.dart`
- `lib/services/employee_service.dart`
- `lib/models/employee_model.dart`

**استمارة الموظف (أمانة السر):**
```
الاسم الكامل *
رقم السجل المدني / الهوية *
الجنسية *
تاريخ الميلاد *
مكان الإقامة
رقم الموبايل *
البريد الإلكتروني
المؤهل العلمي *
الاختصاص *
تاريخ التعيين *
نوع الوظيفة (معلم / إداري / فني / حارس / عامل / أخرى) *
القسم التابع له *
صورة شخصية
ملاحظات
```

**آلية العمل الكاملة:**
```
أمانة السر ← تملأ الاستمارة الشخصية (بدون أي حقول مالية)
          → حفظ
          → الحالة: "بانتظار المراجعة"
          → إشعار للإدارة

الإدارة ← ترى قائمة "بانتظار المراجعة"
        ← تفتح استمارة الموظف
        ← تدخل الصلاحية المالية:
            - الراتب الأساسي (قابل للتعديل لاحقاً)
            - قيمة الساعة
            - عدد ساعات العمل (شهرياً)
            - المكافآت (إن وجدت)
            - الخصومات (إن وجدت)
        ← تقرر:  [✅ قبول]  أو  [❌ رفض]
        
        ✅ قبلت ← الموظف يصبح "نشط"
                 → إشعار لأمانة السر والمحاسبة:
                    "تم تفعيل الموظف (الاسم)"

        ❌ رُفض ← الموظف يبقى "مرفوض"
                 → سبب الرفض (نص اختياري يكتبه المدير)
                 → إشعار لأمانة السر:
                    "تم رفض الموظف (الاسم) - السبب: ..."
```

**بعد التفعيل — صلاحيات المدير الدائمة:**
```
في أي وقت، المدير يقدر يدخل على أي موظف نشط ويعدل:
- زيادة/نقصان الراتب الأساسي
- تغيير قيمة الساعة
- تعديل عدد ساعات العمل
- منح مكافآت مالية إضافية (مكافأة أداء، عيد، مناسبة)
- إضافة خصومات (تأخير، غياب، خصم إداري)

وسجل التعديلات المالية يُحفظ في قاعدة البيانات كسجل تاريخي.
```

**الحقول المالية تظهر فقط في:**
- صفحة المحاسبة (عرض كل الموظفين مع رواتبهم)
- صفحة الإدارة (التعديل)
- **لا تظهر أبداً في أمانة السر**

**إضافة مدخل إيراد جديد:**
```
المبلغ *
التصنيف *
التاريخ *

الموظف المسجل
```

**التصنيفات الافتراضية:**

4. فواتير إنترنت

6. قرطاسية
7. إيجار

**المستخدم يقدر:**
- يضيف تصنيف صرفي جديد
- يحذف تصنيف (إذا لم يكن مستخدماً)

**إضافة صرفية جديدة:**
```
المبلغ *
```

**الإحصائيات:**

### المرحلة 5 — اجتماعات أولياء الأمور (D)

**الملفات:**
- `lib/pages/parent_meetings_page.dart`
- `lib/services/meeting_service.dart`
- `lib/models/meeting_models.dart`

**ميزات الاجتماعات:**
1. إضافة اجتماع جديد:
   - اسم الاجتماع / عنوانه
   - التاريخ
   - الوقت
   - المكان
   - الصفوف المستهدفة (كل الصفوف / صف محدد)
   - ملاحظات

2. تسجيل الحضور:
   - فتح الاجتماع ← تظهر قائمة بأولياء أمور الطلاب
   - لكل ولي أمر:  [✅ حضر]  أو  [❌ غائب]
   - إمكانية تصفّح الطلاب حسب الصف

3. الإحصائيات:
   - نسبة الحضور الإجمالية
   - عدد الحاضرين / عدد الغائبين
   - قائمة الغائبين (للتواصل معهم)

4. التقرير النهائي:
   - كشف حضور كامل (للطباعة)
   - تاريخ الاجتماع، العنوان، النسبة المئوية للحضور
   - توقيع المدير + خاتم المدرسة في المطبوع

5. مؤشر في صفحة الطالب:
   - "ولي الأمر حضر آخر اجتماع؟" ✅ / ❌
   - تاريخ آخر اجتماع حضره ولي الأمر

### المرحلة 10 — النسخ الاحتياطي والاستعادة الحقيقيان (H)

**الملف:** `lib/services/backup_service.dart`

**صيغة النسخة:**
```
ROSE_BACKUP_2026_07_07_143022.zip
```

**محتويات الملف المضغوط:**
```
ROSE_BACKUP_2026_07_07_143022/
├── database/
│   └── rose_school_2026.db
├── files/
│   ├── students/
│   ├── employees/
│   └── school/
├── snapshot.json           ← جميع البيانات بنص JSON
├── manifest.json           ← معلومات النسخة
└── preview.html            ← معاينة سريعة
```

**manifest.json:**
```json
{
  "backupName": "ROSE_BACKUP_2026_07_07_143022",
  "createdAt": "2026-07-07T14:30:22",
  "appVersion": "1.0.0",
  "studentCount": 250,
  "employeeCount": 30,
  "fileCount": 1200,
  "databaseSizeBytes": 12582912,
  "note": "نسخة احتياطية قبل بدء العام الدراسي"
}
```

**الاستعادة:**
1. يختار ملف .zip
2. فحص الملف (تحقق من manifest.json)
3. تأكيد: "سيتم استبدال كل البيانات الحالية"
4. كلمة سر المدير
5. استخراج الملفات ← نسخ الـ DB والملفات
6. تأكيد النجاح
